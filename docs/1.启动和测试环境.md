# 1. 启动和测试环境

## 前置条件

- 真机已连接 ARX LIFT2 + 3 × RealSense D405
- ROS2 已安装并 source
- CAN 接口已配置（CAN1 / CAN3 / CAN5）
- [uv](https://docs.astral.sh/uv/) 已安装（`curl -LsSf https://astral.sh/uv/install.sh | sh`）

> **为什么用 uv？** ROS2 绑定系统 Python，直接用 pip 可能污染 ROS2 环境或版本冲突。uv 管理独立虚拟环境，同时可以继承系统 Python 的 ROS2 包（`rclpy`, `cv_bridge` 等）。

---

## Step 0：克隆仓库

```bash
git clone https://github.com/pickxiguapi/iffyuan-ARX-Toolkit.git
cd iffyuan-ARX-Toolkit

# 拉取 submodule（ARX LIFT2 官方 ROS2 驱动）
git submodule update --init --recursive
```

> `src/LIFT/` 是 submodule，不执行这一步的话该目录为空，后续 ROS2 节点无法启动。

## Step 1：启动 ROS2 节点

在 `ros_scripts/` 目录下运行 `all.sh`，它会依次拉起所有底层节点：

```bash
cd /path/to/iffyuan-ARX-Toolkit/ros_scripts
bash all.sh
```
以上都是终极解决方案，目前还没兼容，现在短期使用这些命令：

```bash
cd /home/arx/Arx_Lift2s/Script
bash all.sh
```

`all.sh` 启动顺序：

| 顺序 | 终端名 | 启动内容 | 说明 |
|------|--------|---------|------|
| 1 | can1 / can3 / can5 | CAN 总线接口 | 硬件通信基础 |
| 2 | body&lift | `arx_lift_controller lift.launch.py` | 底盘 + 升降控制 |
| 3 | left&right | `arx_x5_controller v2_double_arm.launch.py` | 双臂关节控制 |
| 4 | h_camera | RealSense camera_h | 头部相机 (serial: 409122274317) |
| 5 | l_camera | RealSense camera_l | 左侧相机 (serial: 409122272587) |
| 6 | r_camera | RealSense camera_r | 右侧相机 (serial: 409122272707) |

启动后会打开 6 个 gnome-terminal 窗口，等待所有节点就绪（约 5–10 秒）。

### 验证节点是否正常

```bash
# 检查 topic 列表
ros2 topic list

# 应该能看到：
#   /arm_cmd_l, /arm_cmd_r
#   /arm_status_l, /arm_status_r
#   /body_information, /ARX_VR_L
#   /camera_h_namespace/camera_h/color/image_rect_raw
#   /camera_h_namespace/camera_h/aligned_depth_to_color/image_raw
#   /camera_l_namespace/... , /camera_r_namespace/...

# 检查臂状态是否有数据
ros2 topic echo /arm_status_l --once

# 检查相机是否有图像
ros2 topic hz /camera_h_namespace/camera_h/color/image_rect_raw
```

## Step 3：最小可用示例

运行 `scripts/mini_env_example.py`：

```bash
python scripts/mini_env_example.py
```

脚本内容：

```python
from arx_toolkit.env import ARXEnv
import numpy as np

env = ARXEnv(
    action_mode="absolute_eef",
    camera_type="rgbd",
    camera_view=("camera_h",),
    img_size=(640, 480),
)

# reset：双臂回零，升降归零，底盘停止
obs = env.reset()
print("obs keys:", sorted(obs.keys()))
print("left_eef_pos:", obs["left_eef_pos"])
print("left gripper:", obs["left_joint_pos"][6])  # 0=开, 1=闭

# step：左臂移动 + gripper 全开
obs = env.step({
    "left":  np.array([0.1, 0, 0.15, 0, 0, 0, 0.0]),
    "right": None,
    "base":  None,
    "lift":  None,
})

env.close()
```

## Step 3：运行完整测试

节点全部就绪后，运行内置测试脚本：

```bash
python -m arx_toolkit.env.arx_env
```

这会执行 `arx_env.py` 底部的测试脚本，依次测试所有功能。

---

## 内置测试脚本覆盖的功能

`python -m arx_toolkit.env.arx_env` 会按顺序执行以下测试：

| # | 测试项 | 说明 |
|---|--------|------|
| 1 | `reset()` | 双臂回零 + 升降归零 + 底盘停止，打印 obs 结构 |
| 2 | `step()` 双臂 | absolute_eef 模式，双臂同时控制 |
| 3 | `step()` 单臂 | 只控制左臂，右臂 `None` |
| 4 | `step()` gripper 半闭合 | gripper = 0.5 |
| 5 | `step()` gripper 全闭 | gripper = 1.0 |
| 6 | `step_lift()` | 升降到 18 |
| 7 | `step_base()` | 底盘前进 1 秒后停止 |
| 8 | `step_base_lift()` | 底盘旋转 + 升降联合 |
| 9 | `step()` 底盘+升降 | 通过统一 action dict 控制底盘和升降 |
| 10 | `step()` 全 None | 纯取观测，不发任何命令 |
| 11 | `get_observation()` | 局部获取（仅臂 / 仅相机） |
| 12 | `set_mode()` | 左臂重力补偿 → 回零 |
| 13 | `close()` | 安全退出 |

## 常见问题

### 节点启动后 step 报 "no subscribers"

ROS2 节点还没完全就绪。`all.sh` 有 `sleep` 间隔，但实际启动时间取决于硬件。可以先用 `ros2 topic list` 确认所有 topic 都在，再运行环境。

### `import rclpy` 报 ModuleNotFoundError

虚拟环境没有继承系统包。确认创建时用了 `--system-site-packages`：

```bash
# 删掉重建
rm -rf .venv
uv venv --system-site-packages
source .venv/bin/activate
uv pip install -e .
```

同时确认已经 source 了 ROS2 的 setup.bash（在激活虚拟环境**之前**）。

### CAN 报错

确保 CAN 接口已经 up：

```bash
ip link show can1
# 应该看到 state UP
```

如果没有，手动启动：

```bash
sudo ip link set can1 up type can bitrate 1000000
```

### 相机没有图像

检查 RealSense USB 连接：

```bash
rs-enumerate-devices | grep Serial
# 应该看到 409122274317, 409122272587, 409122272707
```

如果只接了部分相机，修改 `camera_view` 参数只订阅已连接的相机：

```python
env = ARXEnv(camera_view=("camera_h",))  # 只用头部相机
```
