# 2-1. 主从遥操作

## 概述

Teleop 模块提供单臂主从遥操作（leader-follower）：人手拖动一只臂（leader），另一只臂（follower）实时镜像跟随。

**工作原理：**

1. Leader 臂进入 gravity 模式（mode=3），人可自由拖动
2. 后台线程以固定频率（默认 50 Hz）循环：读 leader 关节位置 → 低通滤波 → 死带抑制 → 发送给 follower
3. 按 Enter 暂停（follower 保持当前位置），再按 Enter 继续；按空格双臂回零并退出

## 快速开始

```bash
python scripts/teleop_leader_follower.py
```

默认配置：左臂 leader（人拖动），右臂 follower（自动跟随）。

启动后会看到：

```
=== Leader-Follower Teleop [RUNNING] ===
  Leader : left (gravity mode — drag freely)
  Follower: right (mirrors leader)
  Rate   : 50.0 Hz

  [Enter] pause   [Ctrl+C] quit
```

拖动左臂，右臂会实时跟随。

### 按键控制

所有按键为**单键触发**，不需要按回车确认：

| 状态 | 按键 | 效果 |
|------|------|------|
| 运行中 | `Enter` | 暂停，follower 保持当前位置 |
| 运行中 | `Ctrl+C` | 直接退出 |
| 暂停中 | `Enter` | 恢复遥操作 |
| 暂停中 | `Space` | 双臂回零（go home）并退出 |
| 暂停中 | `Ctrl+C` | 直接退出（不回零） |

## 命令行参数

```bash
python scripts/teleop_leader_follower.py [OPTIONS]
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--leader` | `left` | 主臂（`left` 或 `right`），被拖动的那只 |
| `--rate` | `50.0` | 控制循环频率（Hz） |
| `--alpha` | `0.5` | 低通滤波系数，范围 (0, 1]。越小越平滑但延迟越大，1.0 = 不滤波 |
| `--deadband` | `0.004` | 死带阈值（rad），变化量小于此值的关节不更新，抑制微小抖动 |

### 示例

```bash
# 右臂当 leader
python scripts/teleop_leader_follower.py --leader right

# 更平滑（alpha 更小），降低频率
python scripts/teleop_leader_follower.py --alpha 0.3 --rate 30

# 不滤波，直接透传
python scripts/teleop_leader_follower.py --alpha 1.0 --deadband 0.0
```

## Python API

也可以在代码中直接使用：

```python
from arx_toolkit.env import ARXEnv
from arx_toolkit.teleop import LeaderFollowerTeleop

env = ARXEnv(
    action_mode="absolute_joint",
    camera_type="rgb",
    camera_view=(),           # 纯遥操作不需要相机
)

teleop = LeaderFollowerTeleop(
    env,
    leader_side="left",       # 主臂
    control_rate=50.0,         # Hz
    lowpass_alpha=0.5,         # 滤波系数
    deadband=0.004,            # 死带
)

# 方式 1：交互式（阻塞，按 Enter 停止）
teleop.run_interactive()

# 方式 2：手动控制生命周期
teleop.start()
# ... 做其他事情 ...
teleop.stop()

env.close()
```

### API 参考

| 方法 | 说明 |
|------|------|
| `start()` | 设置 leader 为 gravity 模式，启动控制线程 |
| `stop()` | 停止控制线程，follower 保持当前位置 |
| `run_interactive()` | 交互式循环：Enter 暂停/恢复，Space 回零退出 |

## 技术细节

### 控制链路

```
leader 臂 (gravity 模式)
    │
    ▼
env.get_observation()  ──→  leader_joint_pos (7D: 6 关节 + 归一化 gripper)
    │
    ▼
低通滤波: filtered = α × target + (1-α) × previous
    │
    ▼
死带抑制: |filtered - previous| < threshold 的维度不更新
    │
    ▼
env._apply_absolute_joint()  ──→  follower 臂 (joint 模式)
```

### 关节空间控制

遥操作使用 **joint 空间**（`absolute_joint` 模式），直接映射 6 个关节角度 + 1 个夹爪。相比末端执行器空间（EEF），joint 模式更简单直接，无需逆运动学，延迟更低。

### Gripper 归一化

- `get_observation` 返回的 `joint_pos[6]` 已归一化到 [0, 1]（0=全开，1=全闭）
- `_apply_absolute_joint` 接收归一化值，内部自动转换为硬件值 [-3.4, 0.0]
- 滤波在归一化空间进行，无需额外转换

### 滤波参数调优

| 场景 | alpha | deadband | 说明 |
|------|-------|----------|------|
| 精细操作 | 0.3–0.5 | 0.004 | 平滑手部抖动，适合抓取小物体 |
| 快速运动 | 0.7–1.0 | 0.002 | 低延迟，适合大幅运动 |
| 调试 | 1.0 | 0.0 | 无滤波直通，观察原始跟踪效果 |

## 常见问题

### follower 跟随有明显延迟

`alpha` 值太小。尝试增大到 0.7 或 1.0：

```bash
python scripts/teleop_leader_follower.py --alpha 0.7
```

### follower 有轻微抖动

正常情况下死带可以抑制。如果仍然抖动，增大 deadband：

```bash
python scripts/teleop_leader_follower.py --deadband 0.01
```

### leader 臂拖不动

确认 leader 已进入 gravity 模式。如果日志中没有 `Setting left arm to gravity mode` 的输出，检查 ROS2 节点是否正常。

### 停止后 leader 臂仍在 gravity 模式

暂停后按 Space 即可双臂回零。如果用 Ctrl+C 直接退出了，可以手动恢复：

```python
env.set_mode(1, side="both")  # home both arms
```
